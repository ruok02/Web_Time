# Web_Time
3학년 웹 프로그래밍의 사이드 프로젝트(약속 관리 웹 애플리케이션)

웹 프로그래밍 프로젝트 계획서

I. 프로젝트 개요 및 정의 (IT 기획 부분)
1. 프로젝트명
약속을 지키는 습관, [꼬옥!]
→ 지각 패널티 관리 웹 애플리케이션
(위치 공유 기반 서비스까지 추가할 수 있다면 넣을 예정)


2. 개발 목표
사용자가 약속 정보를 저장하고, 페널티를 설정하며, 약속 당일 간단한 체크인 로직을 통해 지각 여부를 판단하는 웹 애플리케이션을 만든다.


3. 서비스 정의
서비스의 목적과 핵심 가치 정의
→ 지각 페널티 기능을 통해 친구들과의 약속을 잊지 않고 즐겁게 지킬수 있도록 돕는 웹 애플리케이션(동적 처리 & 상호작용)
웹 페이지 ↔ 웹 애플리케이션의 차이는 존재함. (→ JavaScript 중요)


4. 기술 스택
HTML, CSS, JavaScript


5. 개발 기간
2025.11.04 ~ 2025.12.08 (5주)


6. 목표 사용자
약속이 많은 10~20대 학생들 및 친구 그룹
?

II. 서비스 구조 및 화면 설계(IT 기획 부분)
1. 메뉴 구조도
서비스의 전체 메뉴 계층 구조를 도식화
Home > 약속 등록, 약속 목록 > 약속 상세, 페널티 내역


2. 정보 구조도(IA)
화면에 노출되는 기능을 계층적으로 표현 (핵심 기능만)
약속 상세 > 약속 정보, D-Day, 참여자 명단, 패널티 내용, 체크인 버튼


3. 주요 기능 명세서
화면별 필요한 핵심 기능 정리
약속 등록: 필수 입력 유효성 검사,
목록: D-Day/지각 상태 표시,
체크인: 현재 시간과 약속 시간 비교 로직


4. 와이어프레임
화면 헤더, 버튼, 목록 등의 위치를 간략하게 스케치


5. 기능 구현
MVP/규모 산정
위치 공유 기능은 일단 제외하고, localStorage 기반의 데이터 관리와 날짜/시간 로직에 집중한다.


6. 기술 선택
불필요한 MSA기술 외부 API 사용은 제외하고, 필요한 JavaScipt의 Date 객체 및 localStorage 활용에 집중함.



III. 개발 및 일정 계획 (정책 부분)
1. 핵심 로직 정의
지각 판단 룰 : 체크인 시각 > 약속 시간 이면 지각 상태 부여


2. 상태 값 정의
서비스에 필요한 상태(Rule)를 미리 정의


3. 주차별 개발 계획
1주차
기획서 완성 및 기본 구조(Layout) 개발
HTML/CSS: 메인, 목록, 등록 폼 페이지 레이아웃 완성. 폰트/색상 등 스타일 가이드 확정. 와이어프레임 기반 UI 구현.

2주차
데이터 저장(CRUD - C/R) 및 목록 표시
약속 등록 폼 데이터 localStorage에 저장 및 목록에 동적으로 불러와 표시하는 로직 구현.

3주차
핵심 로직 구현 (시간/날짜)
D-Day 계산로직 구현 및 목록에 적용. 약속 수정(U), 삭제(D) 기능 구현하여 CRUD 완성.

4주차
핵심 기능 완성 (체크인)
체크인' 버튼 클릭 시 현재 시간 기록 및 약속 시간과의 비교 로직구현. 지각 여부에 따라 목록 상태 및 페널티 내용을 동적으로 변경/표시.

5주차
안정화, 발표 준비 및 시연 자료 제작. 버그 디버깅 및 사용자 테스트. CSS/UI 완성도 최종 검토. 기획서 내용을 기반으로 발표 슬라이드 제작 및 시연 스크립트 작성.
